Sentiment Analysis — BERT vs LLaMA 3.1

NLP
BERT
Generative AI
Python
Hospitality
Comparing a fine-tuned BERT model against zero-shot LLaMA 3.1 on 1,610 real hospitality reviews from four Dublin venues. BERT achieved 97.76% accuracy, LLaMA 98.21% — with no training data.
Author

Ruairí Cullen

Published

April 16, 2026

Overview

This project addresses the business problem of automated sentiment classification of customer reviews across four Dublin venues within the Mercantile Group — NoLIta, Pichet, Café en Seine, and The Mercantile Hotel — using reviews sourced from OpenTable, TripAdvisor, Google, and Yelp.

Two AI models were compared:

  • BERT (bert-base-uncased) — a discriminative transformer model, fine-tuned on labelled review data
  • LLaMA 3.1 — a generative large language model, applied via zero-shot prompting with no training data
Note

This project uses real customer review data from the Mercantile Group. Formal consent was obtained from the General Manager. All personal identifiers were removed prior to analysis.

Key Results

Model Accuracy F1 Score
TF-IDF + Logistic Regression (Baseline) 91.03% 0.952
BERT (Fine-tuned) 97.76% 0.987
LLaMA 3.1 (Zero-shot) 98.21% 0.990
Tip

Key finding: LLaMA 3.1 marginally outperformed fine-tuned BERT despite using zero training data — highlighting the power of large language models for domain-specific NLP tasks with limited labelled datasets.

Dataset

  • 1,610 original records across 4 venues; 1,188 retained after cleaning
  • Binary sentiment labels: Positive (rating ≥ 4) and Negative (rating ≤ 2)
  • Rating 3 deliberately excluded as ambiguous
  • 80/20 stratified train/test split
  • Class imbalance: ~88% positive, ~12% negative

Python Code — Model Comparison Visualisation

import matplotlib.pyplot as plt
import numpy as np

models   = ['TF-IDF + LR\n(Baseline)', 'BERT\n(Fine-tuned)', 'LLaMA 3.1\n(Zero-shot)']
accuracy = [91.03, 97.76, 98.21]
f1       = [95.17, 98.73, 98.98]

x      = np.arange(len(models))
width  = 0.35
colors = ['#475569', '#4C9AFF', '#A78BFA']

fig, ax = plt.subplots(figsize=(9, 5))
bars1 = ax.bar(x - width/2, accuracy, width, label='Accuracy (%)',
               color=colors, edgecolor='white', alpha=0.9)
bars2 = ax.bar(x + width/2, f1, width, label='F1 Score (%)',
               color=colors, edgecolor='white', alpha=0.5)

ax.set_ylabel('Score (%)', fontsize=11)
ax.set_title('Model Comparison — Accuracy & F1 Score', fontsize=13,
             fontweight='bold', pad=15)
ax.set_xticks(x)
ax.set_xticklabels(models, fontsize=10)
ax.set_ylim(85, 102)
ax.legend(fontsize=10)
ax.axhline(y=97, color='grey', linestyle='--', alpha=0.4, label='97% benchmark')

for bar in bars1:
    ax.text(bar.get_x() + bar.get_width()/2, bar.get_height() + 0.2,
            f'{bar.get_height():.1f}%', ha='center', fontsize=9, fontweight='bold')
for bar in bars2:
    ax.text(bar.get_x() + bar.get_width()/2, bar.get_height() + 0.2,
            f'{bar.get_height():.1f}%', ha='center', fontsize=9, fontweight='bold')

plt.tight_layout()
plt.show()

Python Code — BERT Training Loss Curve

import matplotlib.pyplot as plt

epochs = [1.0, 2.0, 3.0]
losses = [0.2025, 0.048, 0.016]

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(epochs, losses, 'r-o', linewidth=2.5, markersize=8, label='Training Loss')
ax.fill_between(epochs, losses, alpha=0.1, color='red')
ax.set_xlabel('Epoch', fontsize=11)
ax.set_ylabel('Loss', fontsize=11)
ax.set_title('BERT Training Loss per Epoch', fontsize=13,
             fontweight='bold', pad=15)
ax.set_xticks([1, 2, 3])
ax.legend(fontsize=10)

for x, y in zip(epochs, losses):
    ax.annotate(f'{y:.4f}', (x, y), textcoords="offset points",
                xytext=(10, 5), fontsize=9, fontweight='bold')

plt.tight_layout()
plt.show()

Methodology

BERT — Discriminative Fine-tuning

  • Tokeniser: bert-base-uncased WordPiece tokenizer
  • Max sequence length: 64 tokens (justified by EDA — majority of reviews under 64 words)
  • Batch size: 8 (optimised for standard laptop GPU constraints)
  • Optimiser: AdamW, learning rate 2e-5
  • Training: 3 epochs with forward pass, backpropagation, and weight updates

LLaMA 3.1 — Zero-shot Prompting

  • Run locally via Ollama API (no cloud dependency — reduced environmental impact)
  • Prompt engineering to constrain output to a single word: Positive or Negative
  • No training data required — leverages generalised pre-trained knowledge

Ethical Considerations

Following the EU Ethics Guidelines for Trustworthy AI, this project ensured:

  • Human oversight — model outputs are decision-support tools, not final decisions
  • Privacy — personal identifiers removed; formal consent obtained from Mercantile Group GM
  • Fairness — equal review counts taken per venue to avoid representation bias
  • Environmental — LLaMA run locally via Ollama to reduce cloud API carbon footprint

Business Implications

Scenario Recommended Model
Limited labelled data available LLaMA 3.1 (zero-shot)
Long-term domain-specific deployment BERT (fine-tuned)
Quick deployment, no data science team LLaMA 3.1
Large dataset, high accuracy priority BERT
